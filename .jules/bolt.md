## 2024-05-24 - PyTorch Scaled Dot Product Attention
**Learning:** PyTorch 2.0+ includes `torch.nn.functional.scaled_dot_product_attention`, which is highly optimized and automatically uses FlashAttention, Memory-Efficient Attention (xFormers), or an efficient math fallback depending on the hardware and inputs. Manual `torch.softmax(q @ k * scale) @ v` is much slower and uses significantly more memory since it instantiates the full `(b, heads, seq_len, seq_len)` attention matrix.
**Action:** Always prefer `F.scaled_dot_product_attention` for attention computations instead of manually calculating the scaled dot product. It speeds up operations and reduces VRAM usage without additional dependencies.

## 2024-05-24 - Precomputing Tensors with `persistent=False`
**Learning:** Precomputing tensors and storing them as buffers can speed up the forward pass significantly. However, using `self.register_buffer('name', tensor)` by default saves the tensor to the module's `state_dict`, which breaks backward compatibility with existing checkpoints if the buffer didn't exist when the checkpoint was created.
**Action:** When adding precomputed tensors (like embedding frequencies) to an existing architecture, always set `persistent=False` in `register_buffer` to avoid saving the buffer to the `state_dict`, preserving backward compatibility while gaining the performance benefit. Additionally, never mutate `self` inside `forward` (e.g. do not write `self.freqs = self.freqs.to(device)`); use a local variable instead (`freqs = self.freqs.to(device)`).
