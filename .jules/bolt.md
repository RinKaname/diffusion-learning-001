## 2024-05-24 - PyTorch Scaled Dot Product Attention
**Learning:** PyTorch 2.0+ includes `torch.nn.functional.scaled_dot_product_attention`, which is highly optimized and automatically uses FlashAttention, Memory-Efficient Attention (xFormers), or an efficient math fallback depending on the hardware and inputs. Manual `torch.softmax(q @ k * scale) @ v` is much slower and uses significantly more memory since it instantiates the full `(b, heads, seq_len, seq_len)` attention matrix.
**Action:** Always prefer `F.scaled_dot_product_attention` for attention computations instead of manually calculating the scaled dot product. It speeds up operations and reduces VRAM usage without additional dependencies.

## 2024-05-22 - [Optimized QKV Splitting and Dimension Extension in Attention/ResBlocks]
**Learning:** QKV splitting via `chunk` + `reshape` on each tensor individually is significantly slower than a unified `view` + `unbind` operation. Using 3D tensors directly for un-headed `F.scaled_dot_product_attention` gracefully avoids redundant dummy head dimensions. Additionally, chaining `.unsqueeze(-1).unsqueeze(-1)` is faster than slicing `[:, :, None, None]` for expanding feature maps, especially in tight bottleneck loops.
**Action:** Always prefer combined `view` operations over splitting-then-reshaping when parsing fused QKV tensors in PyTorch. Prefer `unsqueeze` over `None`/`newaxis` slicing to expand tensor dimensions.
