## 2024-05-24 - PyTorch Scaled Dot Product Attention
**Learning:** PyTorch 2.0+ includes `torch.nn.functional.scaled_dot_product_attention`, which is highly optimized and automatically uses FlashAttention, Memory-Efficient Attention (xFormers), or an efficient math fallback depending on the hardware and inputs. Manual `torch.softmax(q @ k * scale) @ v` is much slower and uses significantly more memory since it instantiates the full `(b, heads, seq_len, seq_len)` attention matrix.
**Action:** Always prefer `F.scaled_dot_product_attention` for attention computations instead of manually calculating the scaled dot product. It speeds up operations and reduces VRAM usage without additional dependencies.

## 2026-06-08 - Precomputing Sinusoidal Embedding Frequencies
**Learning:** In PyTorch models, calculating constant terms like `torch.arange` and `torch.exp` for Sinusoidal embeddings during every `forward` pass wastes CPU/GPU cycles and creates unnecessary ops in the computational graph. Precomputing them is much more efficient. However, they must be stored using `self.register_buffer("freqs", freqs, persistent=False)` so they move correctly to the device, but don't get saved into the model's `state_dict` (which would break backward compatibility with existing checkpoints). Furthermore, one must be careful not to mutate module state (like `self.freqs = self.freqs.to(device)`) inside `forward`.
**Action:** Always precompute static embedding frequencies in `__init__`, register them as non-persistent buffers, and reference them via local variables in `forward` to avoid state mutation anti-patterns.
